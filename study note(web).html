Study Note

[시맨틱 요소]
* w3c에 따르면 "시맨틱 웹을 사용하면 애플리케이션, 기업 및 커뮤니티에서 데이터를 공유하고 재사용할 수 있음."

[스타일 가이드]
* 소문자 요소 이름 사용
* 모든 html 요소 닫기
* 항상 속성 값 인용
* 항상 이미지의 alt, 너비 및 높이 지정
* 긴 코드 줄 피하기, 빈 줄과 들여쓰기
* title 요소 건너 뛰지 않기
* html,body, head 태그 생략 가능
* 소문자 파일 이름 사용

[엔티티]
* 줄 바꿈하지 않는 공백
* 분음 부호 결합(발음 구별 기호)

[문자 집합]
* ASCII 문자 집합 : 제어 문자 0~37(및 127) 값 사용
* ANSI 문자 집합 : 128에서 159 사이의 값에 대한 고유한 문자 집합이 있음
* ISO-8859-1 문자 집합 : 128에서 159 사이의 값을 사용 X
* UTF-8 문자 집합 : 10,000개 이상의 다른 문자로 값 256에서 계속 됨

[URL 인코딩]
* URL = 웹 주소의 다른 단어
* 웹 브라우저는 URL을 사용하여 웹 서버에서 페이지를 요청
* URL 인코딩 : ASCII 문자 집합 을 사용하여 인터넷을 통해서만 보낼 수 있음

[html api]
* 사용자의 지리적 위치를 얻는데 사용
* 오류 및 거부 처리
* getCurrentPosition 메서드 - 데이터 반환
* Geolocation 개체

[css 개요 스타일]
* dotted- 점선 윤곽을 정의
* dashed- 점선 윤곽을 정의
* solid- 실선을 정의
* double- 이중 윤곽선 정의
* groove- 3D 홈이 있는 윤곽을 정의
* ridge- 3D 융기된 윤곽을 정의
* inset- 3D 삽입 윤곽을 정의
* outset- 3D 아웃셋 아웃라인 정의
* none- 윤곽을 정의하지 않음
* hidden- 숨겨진 윤곽선 정의

[css 선택기]
* 단순 선택기(이름, ID, 클래스를 기반으로 요소 선택)
* 결합자 선택기 (요소 간의 특정 관계를 기반으로 요소 선택)
* 유사 클래스 선택기 (특정 상태를 기반으로 요소 선택)
* 유사 요소 선택기 (요소의 일부를 선택하고 스타일 지정)
* 속성 선택기 (속성 또는 속성 값을 기반으로 요소 선택)

[css border 스타일]
* dotted- 점선 테두리를 정의
* dashed- 점선 테두리를 정의
* solid- 단색 테두리를 정의
* double- 이중 테두리 정의
* groove- 3D 홈 테두리를 정의. 효과는 테두리 색상 값에 따라 다름
* ridge- 3D 융기된 테두리를 정의. 효과는 테두리 색상 값에 따라
* inset- 3D 삽입 테두리를 정의. 효과는 테두리 색상 값에 따라 다름
* outset- 3D 아웃셋 테두리를 정의. 효과는 테두리 색상 값에 따라 다름
* none- 경계를 정의하지 않음
* hidden- 숨겨진 테두리를 정의합

[css 박스모델]
* 내용 - 텍스트와 이미지가 표시되는 상자의 내용
* 패딩 - 콘텐츠 주변 영역을 지움. 패딩이 투명
* 테두리 - 패딩과 콘텐츠를 둘러싸는 테두리
* 여백 - 테두리 밖의 영역을 지움. 여백이 투명

[css 드롭다운]
* 사용자가 요소 위로 마우스를 이동할 때 나타나는 상자
* 드롭 다운 메뉴 : 사용자가 목록에서 옵션을 선택할 수 있음
